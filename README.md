## 🧭 The story behind the code

Three years ago, I started writing code to solve a single problem: **why do good people lose time on broken systems?**

Since then, I've replaced unrecorded spreadsheets with a digital platform now controlling **350 keys and 312 movements per month**. I've turned a 10-minute manual contract process into seconds for an operation handling **5,678 clients and ~797 orders per month**. I've migrated abandoned ASP.NET WebForms into modern Next.js platforms that **678 students rely on every day**.

I don't build apps. I build the systems that quietly make people's work possible.

If you're a developer scrolling through profiles wondering whether you'll ever *get there* — I want you to know something:

Three years ago I was where you are. No portfolio, no certainty, just curiosity and the stubborn belief that good systems can make people's lives better. Every line of code on this profile started as a problem I didn't know how to solve.

**The gap between where you are and where you want to be isn't talent. It's reps.**

Build the small thing. Ship the ugly version. Fix it tomorrow. Talk to the people who'll use it. Read the codebase that scares you. The "10x developer" mythology is mostly people who didn't stop showing up.
